The gap

Four stages. Zero governance between them.

Authoring lives in a folder, audit lives in someone's head, the "registry" is a Slack thread, consumption is copy-paste. Each stage is fine in isolation. The chain between them is what's missing.

01 Author

Schema drift goes unnoticed.

Manifests fall out of sync. Capabilities get bumped without semver. No CI step fails on a malformed skill.

02 Audit

Prompt injection ships by default.

Without a static audit, sensitive instructions, secret-leakage patterns, and unsafe tool calls land in production skills that nobody reviewed.

03 Registry

No identity, no rollback.

Skills land in folders, not artifacts. There is no digest to pin, no audit log to replay, no rollback when a release goes wrong.

04 Consume

Five copies, five paths.

Claude, Cursor, Windsurf, Copilot, Kiro — each wants its own format. The same skill drifts five different ways within a week.

Governance

One contract. Every stage of the journey.

Each stage produces a verifiable artifact. The next stage refuses to accept anything else.

01 · author guarantee

Typed at birth.

A scaffolded skill is a manifest, not a folder. Schema, semver, and declared capabilities are required to leave this stage.

→ schema-valid SKILL.md

02 · audit guarantee

Graded before it ships.

9 categories · 35 codes · 70 patterns of static checks, plus LLM-as-judge functional eval. Grade gates the registry push.

→ grade · findings report

03 · registry guarantee

Identity by hash.

SHA-256 content addressing, semver + structural diff, and a hash-chained audit log — tamper-evident across hosts.

→ @sha256:8a2f1c…

04 · consume guarantee

Pinned, never copied.

Developers, agents (via 14 MCP tools), and CI all install by digest. Rollback is one command. Every install lands in the audit log.

→ install --target all

Capabilities

The plumbing behind each stage.

What it actually takes to make the journey enforceable — typed authoring, real audit, content-addressed storage, and pinned consumption across every IDE.

Static security audit

eval audit covers 9 categories with 35 finding codes and 70 regex patterns. --strict adds an AST pass for Python.

Multi-IDE install

One command, five targets. --target all deploys to Claude, Cursor, Windsurf, Copilot, Kiro — each in its native format.

Content-addressed store

apply, get, describe, diff. Every version is a SHA-256 blob with a stable identity.

Provenance & rollback

Every push appends to a hash-chained audit log. install @sha256:… pins to an exact digest; rollback is one command.

Export & import

Portable archives. Move skills between registries, branches, and environments without losing version history.

Claude Code MCP plugin

14 MCP tools and 3 first-class skills give Claude Code a direct line into the registry — list, validate, install, audit.

Quickstart

Sixty seconds, end to end.

Real commands. skillctl doesn't do anything fancy — it just does the boring lifecycle work for you.

~/projects/my-org-skills
# 1. Install the CLI
$ pip install skillsops

# 2. Scaffold a typed skill
$ skillctl create skill my-org/code-reviewer
✓ created skill.yaml, SKILL.md, README.md

# 3. Validate the manifest, capabilities, semver
$ skillctl validate
✓ schema · 0 errors    semver · OK    capabilities · 4 declared

# 4. Run the static security audit
$ skillctl eval audit .
✓ 9 categories · 0 findings · grade A

# 5. Push to the content-addressed store
$ skillctl apply
✓ my-org/code-reviewer@0.1.0  sha256:8a2f1c…

# 6. Deploy to every IDE in one command
$ skillctl install my-org/code-reviewer@0.1.0 --target all
✓ claude · cursor · windsurf · copilot · kiro

CI audit

skillctl eval audit ./skills/ --fail-on-warning --format=github

Self-hosted registry

skillctl serve --hmac-key "$SKILLCTL_HMAC_KEY"

Version diff

skillctl diff my-org/x@1.2.0 my-org/x@1.3.0
